1 # Changes of Note
2 
3 - PDAL's Travis CI configuration is now based on Alpine Linux.
4 - PDAL is now built into containers with Alpine linux in addition to Ubuntu linux. [Tags](https://hub.docker.com/r/pdal/pdal/tags/) exist for each release, starting with 1.4, as well as the master branch.
5 - Pipeline tag names can now contain capital letters. They can also contain underscores after the first character.
6 - Replace `filters.programmable` and `filters.predicate` with the more general `filters.python`.
7 - Add support for Matlab with `filters.matlab`(#1661).
8 - Remove the `approximate` option from `filters.pmf` and add an `exponential` option.p
9 - Placed base64 encoded VLR data in a subnode of the VLR itself with the key "data" rather than duplicate the VLR node itself (#1648).
10 - XML pipelines are no longer supported (#1666).
11 - The number of proprietary dimensions in `readers.text` was expanded from 255 to 4095 (#1657).
12 - API hooks have been added to support the use of PDAL with JVM languages such as Java or Scala.
13 - Added support for LASzip 1.4 and switch to use the new LASzip API. (#1205). LASzip support in PDAL will require LASzip.org release 3.1.0 or greater.
14 - The cpd kernel has been replaced with `filters.cpd`.
15 - No more warnings about ReturnNumber or NumberOfReturns for LAS permuations (#1682).
16 - The KernelFactory class has been removed. Its functionality has been moved to StageFactory.
17 - Built-in eigen support has changed from version 3.2.8 to 3.3.4 (#1681).

19 # Enhancements
20 
21 - API users can now create synonyms for existing arguments to a stage.
22 - `filters.splitter` can now create buffered tiles with the `buffer` option.
23 - `writers.ply`can now be made to write faces of an existing mesh (created with `filters.greedyprojection` or `filters.poisson`) if the `faces` option is used. An option `dims` has also been added that allows specification of the dimensions to be written as PLY elements. The writer also now supports streaming mode.
24 - `readers.text` is now automatically invoked for .csv files.
25 - PDAL_PLUGIN_INSTALL_PATH can now be set via override when building PDAL from source.
26 - Changed the use of null devices to eliminate potentially running out of file descriptors on Windows.
27 - `filters.randomize` can now be created by the stage factory (#1598).
28 - Provide the ability to specify a viewpoint and normal orientation in `filters.normal` (#1638).
29 - `readers.las` now provides the `ignore_vlr` option to allow named VLRs to be dropped when read (#1651).
30 - Allow `writers.gdal` to write output rasters of type other than double (#1497).
31 - `filters.sqlite` is now invoked automatically for .gpkg files.
32 - `filters.colorinterp` can now be used in streaming mode in some cases (#1675).
33 - Pointers can now be stored as metadata.
34 - `filters.ferry` can now create new dimensions without copying data (#1694).

47 # New filters
48 
49 - `filters.greedyprojection` - Performs triangulation of points (surface reconstruction) based on the greedy projection algorithm.
50 - `filters.poisson` - Performs triangulation of points (surface reconstruction) based on the algorithm of Kazhdan.
51 - `filters.head` - Passes through only the first N points.
52 - `filters.tail` - Passes through only the last N points.
53 - `filters.cpd` - Calculates and applies a transformation to align two datasets using the [Coherent Point Drift](https://sites.google.com/site/myronenko/research/cpd) registration algorithm.
54 - `filters.icp` - Calculates and applies a transformation to align two datasets using the [Iterative Closest Point](http://docs.pointclouds.org/trunk/classpcl_1_1_iterative_closest_point.html) registration algorithm.
55 - `filters.voxelcenternearestneighbor` - Finds points closest to the center of a voxel (#1597).
56 - `filters.voxelcentroidnearestneighbor` - Finds points closest to the controid of points in a voxel (#1597).
57 - `filters.python` - Replaces `filters.predicate` and `filters.programmable`.
58 - `filters.matlab` - Provides support for matlab manipulation of PDAL points and metadata (#1661).

61 - Add `readers.osg` to support Open Scene Graph format.
62 - Add `readers.matlab` to support reading data from a user-defined Matlab array struct. The same structure is written by `writers.matlab`.

64 # Bug fixes
65 - Fixed a case where`kernels.tindex` would unconditionally set the spatial reference on a feature from the `a_srs` option. The spatial reference stored in `a_srs` is now only used if explicitly set or no spatial reference was present.
66 - Fixed a case where 'writers.gdal` could fail to check for an out-of-bounds point, potentially leading to a crash.
67 - Fix an error in 'filters.cluster' where the points wouldn't properly be placed in the first cluster because the starting cluster number was incorrect.
68 - Fixed an error in freeing OGR features that could cause a crash when running "pdal density".
69 - Fix potential memory leaks when creating OGRSpatialReference objects.
70 - Make sure the `global_encoding` option is initialized to 0 in `writers.las` (#1595).
71 - Fix eigen::computeCovariance to compute the correct sample covariance.
72 - In some cases, the `filters.crop` would attempt to treat a 2D bounding box as 3D, yeilding a NULL bounding box and an error in behavior (#1626).
73 - Fixed potential crash when using PDAL with multiple threads by providing locking for gdal::ErrorHandler (#1637)
74 - Made sure that an uncompressed LAS file would be properly read even if the `compression` option was provided.
75 - Throw an exception instead of crash when attempting to access a non-existent color ramp. (#1688)
