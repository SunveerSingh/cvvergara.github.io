61 /**
62  A stage performs the actual processing in PDAL. Stages may read data,
63  modify or filter read data, create metadata or write processed data.
64 
65  Stages are linked with setInput() into a pipeline. The pipeline is
66  run with by calling in sequence \ref prepare() and \ref execute() on the
67  stage at the end of the pipeline. PipelineManager can also be used to
68  create and run a pipeline.
69 */

123  /**
124  Execute a prepared pipeline (linked set of stages) in streaming mode.
125 
126  This performs the action associated with the stage by executing the
127  \ref processOne function of each stage in depth first order. Points
128  are processed up to the capacity of the provided StreamPointTable.
129  Not all stages support streaming mode and an exception will be thrown
130  when attempting to \ref execute an unsupported stage.
131 
132  Streaming points can reduce memory consumption, but may limit access
133  to algorithms that need to operate on full point sets.
134 
135  \param table Streming point table used for stage pipeline. This must be
136  the same \ref table used in the \ref prepare function.
137 
138  */
139  void execute(StreamPointTable& table);
