1 Changes of Note:
2 - Command line parsing has been reworked to cause invalid options to emit
3  an error message. Stage options specified in pipelines and on the command
4  line are handled similarly.
5 - The dimension PlatformHeading has been renamed to Azimuth. When looking
6  up a dimension by name, the string "platformheading" is still accepted and
7  returns Azimuth.
8 - Errors detected by GDAL are no longer thrown as exceptions. A log message
9  is emitted instead.
10 - Well-known dimensions are now added to PDAL by editing a JSON file,
11  Dimension.json.
12 - Linking with PDAL using CMake no longer requires explicit linking with
13  curl, jsoncpp or arbiter libraries.
14 - PDAL now searches for plugins in the following locations and order by
15  default: ./lib, ../lib, ../bin, the location where PDAL was installed.
16 - The '--debug' and '--verbose' options are no longer supported as stage
17  options. The '--verbose' option is accepted on the PDAL command line. The
18  '--debug' option is deprecated, and if specified on the command line is
19  equivalent to '--verbose=3'. One can enable logging programmatically by
20  calling setLog() on a PipelineManager or a specific stage.
21 - pdal::Dimension types are now C++11 enumeration classes. The change may
22  require editing any Stage implementations you might have and removing the
23  extraneous ::Enum type specification.
24 
25 Enhancements:
26 - Pipelines can now be read directly from standard input.
27 - Files can now be read from Amazon S3 buckets by providing an appropriate
28  URL.
29 - Many new filters have been added: filters.approximatecoplanar,
30  filters.eigenvalues, filters.estimaterank, filters.hag, filters.normal,
31  filters.outlier, filters.pmf, filters.sample. Most of these are algorithm
32  extractions from the PCL library, with the hope of eliminating the need
33  for PCL in some future PDAL release.
34 - The PLY reader now loads dimensions that aren't predefined PDAL dimensions.
35 - A '--driver' option has been added to allow a specific driver to be loaded
36  for a file without regard to its extension.
37 - The PDAL_DRIVER_PATH environment variable now accepts a list of locations
38  to search for drivers.
39 - Beta release quality drivers improvements in readers.greyhound
40 - Beta quality implementation of Mongus and Zalik ground filter
41 - Experimental implementation of Pingel et al. ground filter
42 - writers.pcd enhancements by Logan Byers (binary, compression) -- requires
43  PCL
44 - Docker images upgraded to Ubuntu Xenial
45 - Cyclone PTS reader -- readers.pts
46 - skewness, kurtosis, stddev, and variance added to filters.stats output
47 - Python API now available https://pypi.python.org/pypi/pdal
48 
49 Fixes:
50 - A failure that may have resulted when using filters.hexbin to calculate
51  density in the southern hemisphere has been corrected.
52 - A failure to create the index file with 'pdal tindex' and GDAL 2.X has
53  been fixed.
54 - The '--tindex' option for the 'pdal tindex' command is now a positional
55  option as specified in the documentation.
56 - The icebridge reader now reads the X dimension as longitude and forces
57  the value in the range (-180, 180]. It also properly uses the dimension
58  Azimuth instead of ScanAngleRank.
59 - An error in writers.pgpointcloud where it ignored SQL to be run at the end
60  of the stage has been fixed.
61 - An error that might incorrectly write values stored internally as bytes
62  when written as a different data type has been fixed.
63 - A problem where 'pdal info' wouldn't properly report dimension names not
64  predefined by PDAL has been fixed.
65 - A bug in filters.crop that wouldn't properly transform coordinates when
66  provided the '--a_srs' option has been fixed.
